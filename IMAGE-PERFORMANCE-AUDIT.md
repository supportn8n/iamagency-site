Исходные PNG сохранены в `public` как рабочие Figma-ассеты, но сайт их больше не запрашивает. Это позволяет вернуться к оригиналу при последующих дизайн-правках без потери качества.

## Производительность

- Для всех Builder/Figma-изображений используются `loading="lazy"` и `decoding="async"`.
- Изображения имеют стабильные размеры в холстах, что снижает CLS.
- Логотипная лента и адаптивная галерея кейсов получили явные `width`/`height`, сохранив CSS-масштабирование.
- Next Image для материалов CMS отдаёт WebP и кэширует оптимизированные варианты 7 дней.
- Для `/blk`, `/hero`, `/partners` и `/utp` добавлен длительный CDN-кэш с суточным browser cache.
- CMS при загрузке новой обложки уже конвертирует её в WebP на стороне редактора.

## Контроль

Повторный аудит:

```powershell
node tools/image-audit.mjs
```

Безопасная lossless-конвертация новых PNG/JPG:

```powershell
node tools/image-audit.mjs --convert
```

Скрипт меняет ссылку только тогда, когда lossless WebP меньше исходного файла. `apple-icon.png` исключена намеренно.
